1. What is the final step after training and validating a machine learning model?
  • Model deployment
  • Reminding the problem statement
  • Data preprocessing
  • Feature extraction

Explanation:

The machine learning lifecycle typically follows these steps: problem definition, data collection, data preprocessing, feature extraction, model selection, model training, model validation, and finally model deployment. After you've trained and validated your model, the natural final step is to deploy it so it can be used in real-world applications.

2. Which metric is specifically used for machine translation evaluation?
  • Word Error Rate
  • BLEU Score
  • Precision
  • Accuracy

Explanation:

BLEU (Bilingual Evaluation Understudy) Score is specifically designed for evaluating machine translation quality. It measures the similarity between machine-generated translations and reference translations by counting matching n-grams. The other options are general metrics not specifically designed for translation tasks.

3. Which method does not relate to data augmentation for noisy text?
  • Back Translation
  • Synonym Replacement
  • Word Embedding
  • Sentence Shuffling

Explanation:

Word Embedding is a technique for representing words as vectors in a continuous space, not a data augmentation technique. Back Translation (translating text to another language and back), Synonym Replacement, and Sentence Shuffling are all valid text augmentation techniques used to create variations of text data.

4. In the context of evaluation, what does ROC-AUC represent?
  • Data transformation method
  • True and false positive rate thresholding
  • Neural network layer
  • Sentence embedding model

Explanation:

ROC (Receiver Operating Characteristic) curve plots the True Positive Rate against the False Positive Rate at various classification thresholds. The Area Under this Curve (AUC) measures the model's ability to distinguish between classes across all possible thresholds. It's not a transformation method, neural network layer, or embedding model.

5. What is SMOTE used for in imbalanced data handling?
  • Data normalization
  • Generating synthetic samples for minority classes
  • Model accuracy evaluation
  • Creating classification models

Explanation:

SMOTE (Synthetic Minority Over-sampling Technique) creates synthetic examples for minority classes by interpolating between existing minority samples. This helps address class imbalance by increasing the representation of minority classes without simply duplicating existing samples.

6. Which technique is used for handling imbalanced data?
  • Outlier Detection
  • Under sampling
  • Cross-validation
  • Data Fusion

Explanation:

Under sampling is a technique specifically designed to handle imbalanced data by reducing the number of samples from the majority class to balance it with the minority class. Outlier detection, cross-validation, and data fusion serve different purposes and aren't primarily for handling imbalanced datasets.

7. What does k-fold cross-validation aim to achieve?
  • High data redundancy
  • Reuse of all data for both training and testing
  • Elimination of validation
  • Onetime testing

Explanation:

K-fold cross-validation divides the dataset into k subsets (folds), then trains and validates the model k times, each time using a different fold as the validation set and the remaining folds as the training set. This allows all data to be used for both training and testing, making efficient use of limited data while reducing the risk of overfitting.

8. What does the F1 Score measure?
  • Data relevance
  • Harmonic mean of precision and recall
  • Linear regression
  • Sum of precision and recall

Explanation:

The F1 Score is calculated as the harmonic mean of precision and recall: 2 × (precision × recall) / (precision + recall). It provides a balanced measure that takes both false positives and false negatives into account, making it useful for imbalanced classification problems.

9. What is the initial step in K-means clustering?
  • Selecting centroids randomly
  • Selecting centroids from the datapoints
  • Determining the optimal number of clusters
  • Calculating distances between points

Explanation:

The standard K-means algorithm begins by randomly selecting K points as initial centroids (not necessarily from the datapoints). It then iteratively assigns points to the nearest centroid and updates the centroid positions until convergence. Determining the optimal number of clusters is typically done before running K-means.

10. Which of the following statements is true about K-means?
  • It always converges to a global minimum
  • It converges to a local minimum
  • It does not converge
  • It uses supervised learning

Explanation:

K-means is guaranteed to converge, but it typically converges to a local minimum of the objective function (sum of squared distances), not necessarily the global minimum. This is why K-means is often run multiple times with different initializations to find better solutions.

1. Dropout and L1/L2 are ________ techniques used to prevent overfitting to noise.

Correct Answer:

regularization

Explanation:

Regularization techniques like Dropout (randomly disabling neurons during training) and L1/L2 regularization (penalizing large weights) help prevent models from becoming too complex and overfitting to training data noise.

2. ________ measures the overall correctness of a model's predictions in classification problem.

Correct Answer:

accuracy

Explanation:

Accuracy is the ratio of correct predictions to total predictions, providing a straightforward measure of overall prediction correctness. However, it can be misleading for imbalanced datasets.

3. Name two hyperparameter tuning approaches.

Correct Answer:

grid search, random search, bayesian optimization, genetic algorithms, gradient-based optimization

Explanation:

Grid Search systematically evaluates all combinations of predefined hyperparameter values, while Random Search samples random combinations, often more efficiently. These are common approaches for finding optimal hyperparameter configurations. Other valid approaches include Bayesian Optimization, Genetic Algorithms, and Gradient-based Optimization.

4. In supervised learning, models are trained using a ________ dataset.

Correct Answer:

labeled

Explanation:

Supervised learning requires labeled data where the target outputs or answers are provided during training. The model learns to map inputs to these known outputs, unlike unsupervised learning where no labels are provided.

5. K-means is sensitive to the initial choice of ________, which can affect its convergence.

Correct Answer:

centroids

Explanation:

The initial placement of centroids can significantly impact K-means' final clustering solution because the algorithm converges to local minima. Different initial centroids can lead to different final clusters.

2. What is the difference between kmeans and kmeans++? What is the benefit of using kmeans++ instead of kmeans?

Model Answer:

K-means uses random initialization of centroids, while K-means++ uses a strategic initialization method that selects initial centroids that are far apart from each other. The benefit of K-means++ is that it tends to find better clusters more consistently and converges faster since it starts with a better initialization, reducing the chance of poor local optima.

Explanation:

The initialization strategy in K-means++ provides a more effective starting point that often leads to more optimal clustering solutions with fewer iterations required for convergence compared to standard K-means.

3. Why is clustering preferred over classification for analyzing web data?

Model Answer:

Clustering is preferred for web data analysis because:

  1. Web data is often unlabeled, making supervised classification difficult
  2. Clustering can discover unknown patterns and groupings in diverse web content
  3. It can adapt to the dynamic and evolving nature of web data without requiring retraining
  4. It helps in organizing vast amounts of heterogeneous web information into meaningful groups
  5. It's useful for exploratory analysis when the categories aren't predefined

Explanation:

The web generates massive amounts of unstructured and unlabeled data continuously. Clustering provides a flexible approach to discover patterns without requiring labeled training data, making it well-suited for web analytics, content organization, and user behavior analysis.

4. Why is naive bayes a popular model in classification, and most of the time we use it as a baseline?

Model Answer:

Naive Bayes is popular and often used as a baseline because:

  1. It's computationally efficient and scales well with large datasets
  2. It requires minimal data preparation and hyper-parameter tuning
  3. It performs surprisingly well despite its "naive" independence assumption
  4. It works well with high-dimensional data like text classification
  5. It's relatively simple to implement and interpret
  6. It can handle missing values and is less prone to overfitting

Explanation:

The simplicity, efficiency, and reasonable performance of Naive Bayes make it an excellent starting point before investing in more complex models. Its performance can also serve as a benchmark to determine if more sophisticated approaches are justified for a given problem.

5. Explain why K-means might converge to a local minimum instead of a global minimum.

Model Answer:

K-means converges to a local minimum rather than a global minimum because:

  1. The algorithm uses an iterative optimization approach that makes locally optimal choices at each step
  2. The initial random placement of centroids strongly influences the final solution
  3. The objective function (sum of squared distances) has multiple local minima in the solution space
  4. K-means lacks a mechanism to escape from poor local minima once it starts converging
  5. The convergence process only guarantees improvement in each iteration, not finding the best possible solution

Explanation:

The optimization landscape for K-means contains many possible solutions (local minima), and the algorithm has no way to "jump" between these solutions to find the global optimum. Instead, it follows a deterministic path from its starting position to the nearest local minimum.

6. What are some methods for handling imbalanced textual data in NLP?

Model Answer:

Methods for handling imbalanced textual data in NLP include:

1. Resampling techniques:

  • Oversampling minority classes (e.g., randomly duplicating samples)
  • Undersampling majority classes
  • SMOTE or other synthetic data generation for text

2. Class weighting:

  • Assigning higher weights to minority classes during model training

3. Data augmentation:

  • Synonym replacement
  • Back-translation
  • Text generation using language models
  • EDA (Easy Data Augmentation)

4. Algorithmic approaches:

  • Ensemble methods like balanced random forests
  • Cost-sensitive learning
  • Anomaly/novelty detection for extremely imbalanced cases

5. Evaluation adjustments:

  • Using appropriate metrics like F1-score, precision-recall curves
  • Stratified sampling for validation

Explanation:

Text data presents unique challenges for addressing class imbalance due to its high dimensionality and semantic complexity. These methods help create more balanced representations or adjust the learning process to pay more attention to underrepresented classes, resulting in models that better capture patterns in minority classes.
