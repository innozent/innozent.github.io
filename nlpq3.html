1. What is the final step after training and validating a machine learning model?
  • Model deployment
  • Reminding the problem statement
  • Data preprocessing
  • Feature extraction

Explanation:

The machine learning lifecycle typically follows these steps: problem definition, data collection, data preprocessing, feature extraction, model selection, model training, model validation, and finally model deployment. After you've trained and validated your model, the natural final step is to deploy it so it can be used in real-world applications.

2. Which metric is specifically used for machine translation evaluation?
  • Word Error Rate
  • BLEU Score
  • Precision
  • Accuracy

Explanation:

BLEU (Bilingual Evaluation Understudy) Score is specifically designed for evaluating machine translation quality. It measures the similarity between machine-generated translations and reference translations by counting matching n-grams. The other options are general metrics not specifically designed for translation tasks.

3. Which method does not relate to data augmentation for noisy text?
  • Back Translation
  • Synonym Replacement
  • Word Embedding
  • Sentence Shuffling

Explanation:

Word Embedding is a technique for representing words as vectors in a continuous space, not a data augmentation technique. Back Translation (translating text to another language and back), Synonym Replacement, and Sentence Shuffling are all valid text augmentation techniques used to create variations of text data.

4. In the context of evaluation, what does ROC-AUC represent?
  • Data transformation method
  • True and false positive rate thresholding
  • Neural network layer
  • Sentence embedding model

Explanation:

ROC (Receiver Operating Characteristic) curve plots the True Positive Rate against the False Positive Rate at various classification thresholds. The Area Under this Curve (AUC) measures the model's ability to distinguish between classes across all possible thresholds. It's not a transformation method, neural network layer, or embedding model.

5. What is SMOTE used for in imbalanced data handling?
  • Data normalization
  • Generating synthetic samples for minority classes
  • Model accuracy evaluation
  • Creating classification models

Explanation:

SMOTE (Synthetic Minority Over-sampling Technique) creates synthetic examples for minority classes by interpolating between existing minority samples. This helps address class imbalance by increasing the representation of minority classes without simply duplicating existing samples.

1. Dropout and L1/L2 are ________ techniques used to prevent overfitting to noise.

Correct Answer:

regularization

Explanation:

Regularization techniques like Dropout (randomly disabling neurons during training) and L1/L2 regularization (penalizing large weights) help prevent models from becoming too complex and overfitting to training data noise.

2. ________ measures the overall correctness of a model's predictions in classification problem.

Correct Answer:

accuracy

Explanation:

Accuracy is the ratio of correct predictions to total predictions, providing a straightforward measure of overall prediction correctness. However, it can be misleading for imbalanced datasets.

3. Name two hyperparameter tuning approaches.

Correct Answer:

grid search, random search, bayesian optimization, genetic algorithms, gradient-based optimization

Explanation:

Grid Search systematically evaluates all combinations of predefined hyperparameter values, while Random Search samples random combinations, often more efficiently. These are common approaches for finding optimal hyperparameter configurations. Other valid approaches include Bayesian Optimization, Genetic Algorithms, and Gradient-based Optimization.

1. Mention three performance metrics for regression and three performance metrics for classification.

Model Answer:

Regression metrics:
- Mean Squared Error (MSE)
- Root Mean Squared Error (RMSE)
- Mean Absolute Error (MAE)
- R-squared (Coefficient of Determination)
- Mean Absolute Percentage Error (MAPE)

Classification metrics:
- Accuracy
- Precision
- Recall (Sensitivity)
- F1 Score
- Area Under ROC Curve (AUC-ROC)
- Specificity
- Matthews Correlation Coefficient (MCC)

Explanation:

Regression metrics measure the distance between predicted and actual continuous values, while classification metrics evaluate the correctness of categorical predictions. The choice of metric depends on the specific problem requirements and data characteristics.

2. What is the difference between kmeans and kmeans++? What is the benefit of using kmeans++ instead of kmeans?

Model Answer:

K-means uses random initialization of centroids, while K-means++ uses a strategic initialization method that selects initial centroids that are far apart from each other. The benefit of K-means++ is that it tends to find better clusters more consistently and converges faster since it starts with a better initialization, reducing the chance of poor local optima.

Explanation:

The initialization strategy in K-means++ provides a more effective starting point that often leads to more optimal clustering solutions with fewer iterations required for convergence compared to standard K-means.

3. Why is clustering preferred over classification for analyzing web data?

Model Answer:

Clustering is preferred for web data analysis because:

  1. Web data is often unlabeled, making supervised classification difficult
  2. Clustering can discover unknown patterns and groupings in diverse web content
  3. It can adapt to the dynamic and evolving nature of web data without requiring retraining
  4. It helps in organizing vast amounts of heterogeneous web information into meaningful groups
  5. It's useful for exploratory analysis when the categories aren't predefined

Explanation:

The web generates massive amounts of unstructured and unlabeled data continuously. Clustering provides a flexible approach to discover patterns without requiring labeled training data, making it well-suited for web analytics, content organization, and user behavior analysis.
